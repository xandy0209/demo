# 需求背景
## 需求背景
在政企客户的日常运维保障中，客户投诉和故障事件的处理效率直接影响客户满意度。当前缺乏一个统一的、高效的投诉支撑平台来集中管理工单的发起、处理、查询和配置，导致信息流转不畅、处理进度难以追踪、统计分析困难。因此，需要建立一个综合性的“投诉支撑”模块，实现工单的全生命周期管理。

## 业务场景
1. **场景一：投诉录入与派发**
   客服人员接到客户投诉电话，需要快速录入投诉信息。系统需支持通过产品实例快速回填客户资料，并支持语音输入投诉内容。录入完成后，根据业务类型（如数据专线）支持 A/Z 端的分支派发。
2. **场景二：主动监控转派**
   系统自动识别到网络故障并生成故障事件。调度人员在“故障派单”页面查看事件详情（含快照），一键触发派单，系统自动带入事件关联的业务信息和附件。
3. **场景三：工单处理与反馈**
   运维人员在“待办工单”中查看任务。对于数据专线，A 端和 Z 端处理人分别进行接单、现场排查、回单。回单时需记录故障类型、处理结果及详细原因。
4. **场景四：流程质检与归档**
   质检人员对已回单的工单进行审核。若处理不达标则驳回至处理环节；若通过则工单自动归档。
5. **场景五：管理配置与分析**
   管理人员通过“时限配置”定义不同等级业务的 SLA 要求，通过“客响人员管理”维护一线人员库，并通过“统计分析”看板监控各地市的处理效率和业务可用率。

## 业务对象
1. **工单 (Ticket)**
   记录投诉或故障的全生命周期信息。核心状态包括：T0(待受理)、T1(处理中)、T2(待质检)、Closed(已归档)。
2. **故障事件 (Fault Event)**
   由监控系统自动触发的原始告警记录，可作为工单的来源。
3. **SLA 配置 (SLA Config)**
   定义业务处理的时限标准，分为通用规则（按等级）和个性化规则（按清单）。
4. **客响人员 (Personnel)**
   执行工单处理的一线运维人员信息。

# 需求概述
“投诉支撑”模块旨在提供端到端的工单管理能力，涵盖工单发起、工单处理、综合查询及配置管理四大核心功能。系统通过标准化的流程引擎（支持专线双端分支）和智能化的辅助手段（语音识别、快照预览），提升政企客户故障响应的专业性和及时性。

## 功能说明

### 全局布局与导航
**界面元素:**
1. **左侧菜单栏**
   - 控件类型: 导航树
   - 功能: 包含四个功能分组，支持展开/折叠。点击末级菜单在右侧打开页签。
   - 交互: “待办工单”和“故障派单”支持数字角标实时显示任务数。
2. **顶部页签栏**
   - 控件类型: 页签 (Tabs)
   - 功能: 支持多页签切换、右键关闭（关闭当前/其他/右侧/全部）。

### 新增工单
**界面元素:**
1. **业务分类**
   - 控件类型: 下拉选择
   - 选项: 专线、企宽
2. **产品实例**
   - 控件类型: 文本框（点击触发弹窗）
   - 功能: 点击触发 **【业务查询选择弹窗】**。
     - **【业务查询选择弹窗】详细描述**:
       - **查询条件**: 关键字（客户名称/编号/实例/地址）、业务类型、保障等级、地市。
       - **列表展示**: 客户名称、客户编号、业务类型、产品实例、电路代号、A/Z端保障等级、A/Z端地市及地址。
       - **操作**: 选中行后点击“确认选择”，自动回填表单。
3. **投诉内容**
   - 控件类型: 多行文本框
   - 辅助功能: “语音输入”按钮。点击后调用麦克风，将语音实时转为文字填充。
4. **派单信息**
   - 控件类型: 复选框 + 下拉选择
   - 逻辑: 专线业务支持勾选“派发A端”和“派发Z端”，并分别选择下派对象和班组。

### 故障派单
**界面元素:**
1. **查询区域**: 关键字输入、识别结果（全部/业务中断/保护降级）、产生时间范围。
2. **数据列表**:
| 序号 | 列标题 | 数据字段 | 说明 |
|---|---|---|---|
| 1 | 事件编号 | ticketNo | |
| 2 | 客户名称 | customerName | |
| 3 | 识别结果 | faultType | |
| 4 | 事件快照 | faultSnapshot | 点击“查看快照”触发 **【快照预览弹窗】** |
| 5 | 操作 | - | 点击“派单”跳转至新增工单并带入数据 |

### 待办工单
**界面元素:**
1. **查询区域**: 关键字、工单状态（待受理/处理中/待质检）、保障等级、业务类型。
2. **操作逻辑**:
   - **导出**: 点击触发 **【导出确认弹窗】**，输入文件名后下载 Excel。
   - **处理**: 点击“受理/处理/质检”进入详情视图。

### 工单详情与处理视图
**界面结构:**
1. **基本信息**: 展示客户、业务及派单详情。支持在处理环节修改关联业务。
2. **流转信息**:
   - **流程图**: 
     - 普通业务: 派发 -> 处理 -> 质检 -> 归档。
     - 数据专线: 派发 -> 工单处理[A端地市] & 工单处理[Z端地市] -> 工单质检 -> 归档。
   - **处理日志**: 列表展示操作时间、操作名、环节（格式：环节名称[地市]）、操作人、处理信息。
3. **工单处理**:
   - **T1 环节**: 需录入故障类型、处理结果、故障原因。支持“驳回”和“处理回单”。
   - **T2 环节**: 录入质检结果（通过/驳回）及意见。

### 时限配置
**界面布局:**
页面顶部提供“通用配置”和“个性化配置”两个子页签。

**一、通用配置**
管理基于业务分类和保障等级的标准化时限。
1. **新增规则区域**
   - **业务分类**: 单选按钮组（专线、企宽）。
   - **配置条件**: 
     - 专线模式下显示：AAA、AA、A、普通（多选复选框）。
     - 企宽模式下显示：统付、个付（多选复选框）。
   - **处理时限(小时)**: 数字输入框。
   - **添加规则按钮**: 点击校验必填项，成功后加入下方列表。
2. **数据列表区域**
| 序号 | 列标题 | 说明 |
|---|---|---|
| 1 | 业务分类 | 专线 / 企宽 |
| 2 | 配置条件 | 展示选中的等级或付费类型标签 |
| 3 | 处理时限 (小时) | 对应的 SLA 要求时间 |
| 4 | 配置时间 | 规则创建或最后修改时间 |
| 5 | 配置人 | 操作账号信息 |
| 6 | 操作 | 包含“编辑”和“删除”图标按钮 |
3. **操作逻辑详述**
   - **编辑**: 点击触发 **【通用规则编辑弹窗】**。
     - **【通用规则编辑弹窗】内联描述**:
       - 业务分类为只读。
       - 配置条件支持重新勾选。
       - 处理时限支持重新输入。
       - 底部提供“取消”和“保存修改”按钮。
   - **删除**: 点击触发确认气泡，确认后移除规则。

**二、个性化配置**
针对特定业务清单设置特殊时限。
1. **导入配置区域**
   - **下载模版**: 点击下载 Excel 导入模版。
   - **导入清单**: 点击触发文件选择，仅支持 .xlsx 格式。上传后显示文件名及删除图标。
   - **处理时限(小时)**: 数字输入框。
   - **保存配置按钮**: 点击将清单与时限关联并保存。
2. **数据列表区域**
| 序号 | 列标题 | 说明 |
|---|---|---|
| 1 | 配置业务清单 | 显示“查看业务清单”链接，点击触发 **【名单详情弹窗】** |
| 2 | 处理时限 (小时) | |
| 3 | 配置时间 | |
| 4 | 配置人 | |
| 5 | 操作 | 包含“删除”图标按钮 |
3. **操作逻辑详述**
   - **查看业务清单**: 点击触发 **【名单详情弹窗】**。
     - **【名单详情弹窗】详细描述**:
       - **表格列定义**:
| 序号 | 列标题 | 说明 |
|---|---|---|
| 1 | 复选框 | 用于批量操作 |
| 2 | 产品实例 | 业务唯一标识 |
| 3 | 业务类型 | |
| 4 | 客户名称 | |
| 5 | 客户编号 | |
| 6 | A端保障等级 | |
| 7 | Z端保障等级 | |
| 8 | A端地市 | |
| 9 | Z端地市 | |
| 10 | 操作 | “删除”按钮，移除单条业务 |
       - **批量操作**: 勾选多行后，点击底部“批量删除”按钮，统一移除。
       - **分页**: 支持切换每页显示条数及翻页。

### 客响人员管理
维护一线运维人员库，支持快速检索和批量维护。
**界面元素:**
1. **查询筛选区域**
   - **业务分类**: 下拉选择（全部、专线、企宽）。
   - **关键字**: 文本输入框，支持按姓名或电话模糊搜索。
   - **地市**: 下拉选择内蒙古 12 个地市。
   - **操作按钮**: “查询”执行过滤，“重置”清空条件。
2. **数据列表区域**
| 序号 | 列标题 | 说明 |
|---|---|---|
| 1 | 复选框 | 用于批量删除 |
| 2 | 姓名 | |
| 3 | 电话 | |
| 4 | 业务分类 | |
| 5 | 地市 | |
| 6 | 操作 | 包含“编辑”和“删除”图标按钮 |
3. **底部工具栏**
   - **导入模版**: 下载 CSV 格式的导入模版。
   - **导入**: 触发文件选择，解析 CSV 数据并追加至列表。
   - **导出**: 导出当前列表数据为文件。
   - **新增人员**: 点击触发 **【人员信息编辑弹窗】**。
   - **批量删除**: 仅在选中记录时启用，点击触发确认弹窗。

**操作逻辑详述:**
1. **新增/编辑人员**: 触发 **【人员信息编辑弹窗】**。
   - **【人员信息编辑弹窗】内联描述**:
     - **字段**: 姓名(必填)、联系电话(必填)、业务分类(下拉必选)、地市(下拉必选)。
     - **校验**: 保存时检查必填项，电话号码需符合格式。
2. **删除操作**:
   - 点击单行删除或批量删除，均触发 **【删除确认弹窗】**。
   - **【删除确认弹窗】详细描述**: 提示“确定要删除选中的 X 条客响人员记录吗？此操作不可恢复。”，点击“确定删除”执行操作。

# 非功能性需求
## 性能需求
1. 列表加载时间不超过 1.5 秒。
2. 导入 1000 条人员数据处理时间不超过 3 秒。
## 安全需求
1. 敏感操作（如删除配置、修改工单）必须记录审计日志。
2. 权限控制：仅具备管理权限的用户可进入“配置管理”模块。
## 兼容性需求
1. 适配主流浏览器（Chrome 90+, Edge, Firefox）。
2. 支持 1920*1080 及以上分辨率显示。
